# Curation Checklist: What Must Be Curated vs. Linked

## Must Curate (for agent automation, Reviewer passes, and robust code)
- Key requirements: Explicit, actionable rules or standards for each domain (e.g., error handling, file IO, validation)
- Canonical code patterns: Proven code snippets, templates, or workflows
- Troubleshooting steps: Common errors, root causes, and fixes
- Summaries: Why a repo or pattern matters, and when to use it
- Known-good artifacts: Example outputs, logs, or files for validation
- Version/date stamps: Track when requirements or patterns were last updated

## Can Link (for navigation, but not sufficient for automation)
- Raw URLs to repos, docs, or external resources
- Indexes or lists of related resources
- General overviews without actionable detail

**Note:** Agents will always prioritize curated entries for code, review, and planning. URLs and links are for human navigation and as a fallback for new research.
